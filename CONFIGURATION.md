## `access`

Type: `public | internal`  
Default: `public`

Access level for generated declarations.

<details>
<summary>Example</summary>

Default:
```swift
public struct User: Codable {}
```

Option:
```yaml
access: internal
```

Result:
```swift
struct User: Codable {}
```

</details>

## `acronyms`

Type: `[String]`  
Default: `[url, id, html, ssl, tls, https, http, dns, ftp, api, uuid, json]`

Acronyms to preserve in generated Swift names. For example, `image_url` becomes `imageURL`.

<details>
<summary>Example</summary>

Default:
```swift

```

Option:
```yaml
acronyms:
  - id
```

Result:
```swift
public var imageUrl: String
```

</details>

## `generate`

Type: `[entities | paths | package]`  
Default: `[entities, paths]`

Selects which outputs to write.

- `entities` writes schema models.
- `paths` writes request builders.
- `package` writes a `Package.swift` and places generated sources under `Sources/`.
  - [`module`](#module) must not be empty
  - **Note:** Only provided as a template and only recommended for initial generation.

> [!IMPORTANT]
> This generator does not remove existing files. To ensure proper generation within a pre-existing package/folder, delete the prior output as part of your own generation steps.

<details>
<summary>Example</summary>

Default:
```text
Entities/User.swift
Paths/GetUser.swift
Extensions/Paths.swift
```

Option:
```yaml
module: MyAPI
generate:
  - entities
  - paths
  - package
```

Result:
```text
Package.swift
Sources/Entities/User.swift
Sources/Paths/GetUser.swift
Sources/Extensions/Paths.swift
```

</details>

# Comments

## `comments.annotateDeprecations`

Type: `annotation | comment | none | remove`  
Default: `annotation`

Controls how deprecated schemas, properties, and operations are marked.

- `annotation` emits `@available(*, deprecated, message: "Deprecated")`.
- `comment` emits a documentation warning.
- `none` emits no deprecation marker.
- `remove` skips deprecated schemas, properties, and operations.

> [!CAUTION]
> Removing values may cause compilation or API breakage. Ensure that removing deprecated values is okay with your API.

<details>
<summary>Example</summary>

Default:
```swift
public struct User: Codable {
    public var name: String

    @available(*, deprecated, message: "Deprecated")
    public var legacyName: String?
}
```

Option:
```yaml
comments:
  annotateDeprecations: remove
```

Result:
```swift
public struct User: Codable {
    public var name: String
}
```

</details>

## `comments.options`

Type: `[title | description | capitalized]`  
Default: `[title, description, capitalized]`

Controls generated documentation comments.

- `title` includes schema or operation titles.
- `description` includes schema or operation descriptions.
- `capitalized` capitalizes generated comment text.

Use an empty array to disable generated comments.

<details>
<summary>Example</summary>

Default:
```swift
/// User
///
/// A person account.
public struct User: Codable {}
```

Option:
```yaml
comments:
  options:
    - description
```

Result:
```swift
/// A person account.
public struct User: Codable {}
```

</details>

## `entities.automaticIdentifiable`

Type: `Bool`  
Default: `false`

Adds `Identifiable` to entities that have an `id` property.

<details>
<summary>Example</summary>

Default:
```swift

    public var id: String
}
```

Option:
```yaml

  automaticIdentifiable: true
```

Result:
```swift

}
```

</details>

## `entities.codingStrategy`

Type: `stringCodingKey | codingKeys`  
Default: `stringCodingKey`

Chooses how custom coding keys are generated.

- `stringCodingKey` decodes with a shared `StringCodingKey` helper.
- `codingKeys` writes a nested `CodingKeys` enum.

<details>
<summary>Example</summary>

Default:
```swift
public init(from decoder: Decoder) throws {
    let values = try decoder.container(keyedBy: StringCodingKey.self)
    self.firstName = try values.decode(String.self, forKey: "first_name")
}
```

Option:
```yaml
entities:
  codingStrategy: codingKeys
```

Result:
```swift
enum CodingKeys: String, CodingKey {
    case firstName = "first_name"
}

public init(from decoder: Decoder) throws {
    let values = try decoder.container(keyedBy: CodingKeys.self)
    self.firstName = try values.decode(String.self, forKey: .firstName)
}
```

</details>

# Paths

## `paths.style`

Type: `operations | rest`  
Default: `operations`

Controls the shape of generated request builders.

- `operations` writes a plain list of operation-id based members, such as `Paths.listPets(limit: nil)`.
- `rest` writes CreateAPI-style nested path components, such as `Paths.pets.get(limit: nil)` and `Paths.pets.petID("1").get`.

<details>
<summary>Example</summary>

Default:
```swift
Paths.listPets(limit: nil)
Paths.showPetByID(petID: "1")
```

Option:
```yaml
paths:
  style: rest
```

Result:
```swift
Paths.pets.get(limit: nil)
Paths.pets.petID("1").get
```

</details>

## `paths.bodyTypeOverrides`

Type: `[String: String]`  
Default: `[:]`

<details>
<summary>Example</summary>

</details>

## `paths.inlineQueryParameterLimit`

Type: `Int?`  
Default: `2`

Maximum number of query parameters to inline into path function parameters. If there are more query parameters, then a parameter type will be generated. Set to `null` to always generate a parameter type.

<details>
<summary>Example</summary>

Default:
```swift
public static func searchUsers(term: String, limit: Int) -> Request<SearchUsersResponse> {
    Request(path: "/users/search", method: "GET", query: [("term", term), ("limit", String(limit))], id: "searchUsers")
}
```

Option:
```yaml
paths:
  inlineQueryParameterLimit: null
```

Result:
```swift
public static func searchUsers(parameters: SearchUsersParameters) -> Request<SearchUsersResponse> {
    Request(path: "/users/search", method: "GET", query: parameters.asQuery, id: "searchUsers")
}

public struct SearchUsersParameters {
    public var term: String
    public var limit: Int
}
```

</details>

## `paths.inlineSimpleRequests`

Type: `Bool`  
Default: `true`

Inlines simple request body values directly into path function parameters.

<details>
<summary>Example</summary>

Default:
```swift
public static func updateName(name: String) -> Request<User> {
    Request(path: "/users/name", method: "POST", body: ["name": name], id: "updateName")
}
```

Option:
```yaml
paths:
  inlineSimpleRequests: false
```

Result:
```swift
public static func updateName(_ body: UpdateNameRequest) -> Request<User> {
    Request(path: "/users/name", method: "POST", body: body, id: "updateName")
}
```

</details>

```yaml

```

```swift
    }

```

</details>

## `paths.removeRedundantPaths`

Type: `Bool`  
Default: `true`

When `paths.style` is `rest`, skips common leading path components when every generated path shares the same prefix and doing so does not hide an operation or a path parameter.

<details>
<summary>Example</summary>

Default with paths `/api/v1/users` and `/api/v1/teams`:
```swift

Paths.teams.get
```

Option:
```yaml
paths:
  style: rest
  removeRedundantPaths: false
```

Result:
```swift
Paths.api.v1.users.get
Paths.api.v1.teams.get
```

</details>

## `paths.responseTypeOverrides`

Type: `[String: String]`

## `rename.properties`

<details>
<summary>Example</summary>

```swift
public struct User: Codable {
    public var firstName: String

    public init(from decoder: Decoder) throws {
        let values = try decoder.container(keyedBy: StringCodingKey.self)
        self.firstName = try values.decode(String.self, forKey: "given_name")
    }
}
```

</details>
